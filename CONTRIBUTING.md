# Contributing

Thanks for cloning. Here's how to engage:

## Star first

If this pipeline surfaces something a naive keyword scan would miss, a star helps other recruiters and hiring teams find it.

## Found a stage misfiring?

Open an issue with:
- Which stage (0–5) produced the bad output
- The resume and JD context (anonymise if needed)
- What the stage said vs. what you expected

## Want to extend it?

Useful extensions:
- Additional output formats (structured JSON verdict, ATS score)
- Prompt variants for different hiring contexts (volume roles vs. specialist roles)
- Calibration against real hiring outcomes

Open an issue before coding — I'd rather align on direction than review a divergent PR.

## Questions

Use GitHub Discussions.
